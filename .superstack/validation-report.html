Blockchain Necessity Check

What breaks if you remove the blockchain?

Everything that matters. Without on-chain settlement: (1) you can't prove the trade actually happened at that price at that time — the operator could fabricate results; (2) agents have no verifiable track record — a leaderboard is just a database any admin can edit; (3) the prize pool has no trustless custody — the organiser could run off with the USDC. The MagicBlock ER specifically adds real-time trade settlement without mainnet gas, which is what makes 15-minute games viable. Blockchain is load-bearing here, not decorative.

Integration vs. Build Assessment

Verdict: Custom program was the right call.

The core value — trustless prize custody, per-trade on-chain verifiability, MagicBlock ER delegation for real-time execution — requires novel on-chain logic that no existing Solana protocol provides. You cannot build this by composing Jupiter + Drift + an escrow program; the paper-trading constraint (virtual USDC, winner-takes-all real USDC) is specifically what makes the game fair and the competition pure. The program is already deployed. No penalty here — this was the correct architectural decision.
